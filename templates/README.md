# Templates

Authoring templates for documents in this repository.

## Document templates

- [adr.md](adr.md) — Architecture Decision Record
- [_markdown.md](_markdown.md) — Markdown style guide and conventions

## Issue templates

Issue templates live in the [.github](https://github.com/uips-browser-kit/.github) repository
and apply across the whole organization.

- [Task](https://github.com/uips-browser-kit/.github/blob/main/.github/ISSUE_TEMPLATE/task.yml) — a concrete piece of work with a clear done state
- [Bug Report](https://github.com/uips-browser-kit/.github/blob/main/.github/ISSUE_TEMPLATE/bug_report.yml) — something is not working as expected
- [Feature Request](https://github.com/uips-browser-kit/.github/blob/main/.github/ISSUE_TEMPLATE/feature_request.yml) — propose an addition; scope must be confirmed before implementation
- [Spike](https://github.com/uips-browser-kit/.github/blob/main/.github/ISSUE_TEMPLATE/spike.yml) — time-boxed research with a defined output
- [Scope Input](https://github.com/uips-browser-kit/.github/blob/main/.github/ISSUE_TEMPLATE/scope-input.yml) — browser automation pain point, use case, or scope proposal
- [Documentation Gap](https://github.com/uips-browser-kit/.github/blob/main/.github/ISSUE_TEMPLATE/documentation_gap.yml) — something missing, wrong, or unclear in the docs

New issues are filed directly in the relevant repository using the links above,
or via the Issues tab on any repository in the organization.
